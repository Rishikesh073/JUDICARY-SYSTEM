# LexAgent: Semantic Search Accuracy & Accountability Notes

## 1. The "Different Wording" Problem (Semantic vs. Keyword Search)
LexAgent uses ChromaDB with a Sentence Transformer embedding model (`all-MiniLM-L6-v2`). This means it performs **Semantic Search (Vector Search)** rather than exact keyword matching.
- When a lawyer searches for "medical negligence", the system converts the *meaning* of that phrase into a vector.
- If a case uses the wording "doctor's dereliction of duty", its meaning is also converted into a vector.
- Because the underlying context is the same, their vectors will be close together in the database.
- **Conclusion:** The system will successfully find relevant cases even if the exact keywords differ, because it understands the conceptual meaning.

## 2. How Accuracy is Calculated
Accuracy is calculated mathematically using "Distance Metrics" (e.g., Cosine Similarity or L2 distance). ChromaDB calculates the distance between the query's vector and every case's vector in the database. The cases with the shortest distance (highest similarity) are ranked as the most accurate results.

## 3. The Accountability Problem (Preventing Irrelevant Cases)
Currently, `agent.py` pulls the top 3 results (`n_results=3`) regardless of how relevant they actually are. If the database lacks cases on the queried topic, it will simply return the "least bad" irrelevant cases.
